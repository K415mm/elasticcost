## 🇬🇧 ENGLISH SECTION

### 1. Agent Overview

The **RG-MDR Agent** is the human-intelligence layer of the RaiseGuard MDR 360° platform. It transforms your security posture from passive monitoring to **active, expert-driven defense**. Where automated SIEM rules detect patterns, the RG-MDR agent provides the **dedicated SOC analyst team** that investigates, validates, hunts, and responds to threats in real time.

This agent is purpose-built for environments that face **advanced and persistent threats** — specifically assets that process sensitive data, control network access points, or integrate third-party security tools like XDR platforms. The RG-MDR service layer sits *on top of* the RG-SIEM data foundation, adding human judgment and proactive security engineering.

#### 2.1 Alert Triage & Validation

The RG-MDR team applies **expert human judgment** to every alert generated by the SIEM engine:

- **True Positive confirmation**: Analysts investigate context, confirm attack patterns, and escalate real incidents
- **False Positive suppression**: Continuously tune detection rules to reduce noise without missing real threats
- **Priority scoring**: Apply MITRE ATT&CK framework mapping to assign severity and urgency to confirmed detections

#### 2.2 Proactive Threat Hunting

Unlike reactive monitoring, the RG-MDR service performs **scheduled threat hunts**:

| Hunt Type | Frequency | Methodology |
|---|---|---|
| **IOC Sweep** | Daily | Search for known bad indicators across all log sources |
| **Behavioral Anomaly Hunt** | Weekly | Investigate ML anomaly detections not yet escalated |
| **Lateral Movement Hunt** | Bi-weekly | Map inter-system authentication flows for unusual paths |
| **Persistence Mechanism Hunt** | Monthly | Search for scheduled tasks, registry keys, startup items |
| **Network Exfiltration Hunt** | Monthly | Analyze DNS, proxy, and firewall logs for data exfil patterns |

#### 2.3 Incident Response

When a confirmed incident is detected, the RG-MDR team executes a **structured IR process**:

```
DETECTION  →  ANALYSIS  →  CONTAINMENT  →  ERADICATION  →  RECOVERY  →  LESSONS LEARNED
    │              │               │               │              │              │
  Alert          Full          Block IP /        Remove       Restore        Update
 validated    investigation   isolate host     malware      services        rules
```

**SLA Commitments**:

| Severity | Initial Response | Analyst Assignment | Client Notification |
|---|---|---|---|
| **Critical (P1)** | < 15 minutes | Immediate | < 30 minutes |
| **High (P2)** | < 1 hour | < 2 hours | < 4 hours |
| **Medium (P3)** | < 4 hours | < 8 hours | Next business day |
| **Low (P4)** | < 24 hours | Weekly batch | Weekly report |

#### 2.4 Security Engineering & Configuration

The RG-MDR service includes **active security engineering** work:

- **Network Security**: Review and optimize firewall rules, IDS/IPS signatures, and VPN policies
- **Third-Party Tool Integration**: Onboard and maintain XDR platforms (SentinelOne, Microsoft Defender, CrowdStrike) into the Elastic SIEM pipeline
- **Threat Intelligence Integration**: Maintain and update IOC feeds from commercial and open-source intelligence sources
- **Playbook Development**: Create and maintain automated response playbooks for recurring incident types
